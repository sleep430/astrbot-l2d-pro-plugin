# 平台支持矩阵

本文档说明 `astrbot-live2d-desktop` 在不同桌面平台上的当前支持情况，帮助你在发布、排障和用户答疑时快速判断能力边界。

## 总览

| 能力 | Windows | macOS | Linux (X11) | Linux (Wayland) |
| --- | --- | --- | --- | --- |
| 应用安装包 | 支持（NSIS / Portable） | 支持（DMG / ZIP） | 支持（AppImage / deb / rpm） | 支持（AppImage / deb / rpm） |
| 自动更新 | 支持 | 支持 | 不支持 | 不支持 |
| 透明悬浮主窗口 | 支持 | 支持 | 依赖桌面环境/窗口管理器 | 依赖桌面环境/合成器 |
| 始终置顶 | 支持 | 支持（`screen-saver` 层级） | 支持（效果受桌面环境影响） | 支持（效果受合成器影响） |
| 智能鼠标穿透 | 支持 | 支持 | 不支持 | 不支持 |
| 完整全屏应用检测 | 支持（`node-window-manager`） | 启发式支持（`active-win`） | 启发式支持（`active-win`） | 不支持 |
| Live2D 模型渲染 | 支持 | 支持 | 支持 | 支持 |
| 桌面截图工具 | 支持 | 支持 | 支持 | 支持 |
| 全局录音快捷键 | 支持 | 支持 | 支持 | 支持 |

## 平台差异

### Windows

- 体验最完整。
- 透明窗口、置顶、智能穿透、全屏应用检测均为当前最佳实现。
- 自动更新支持安装版；便携版不支持自动更新。
- `win32 arm64` 当前会禁用原生全屏窗口检测，以避免不稳定行为。

### macOS

- 透明窗口、置顶和快捷键可用。
- 全屏应用检测依赖 `active-win` 的启发式窗口边界判断，稳定性弱于 Windows。
- 自动更新可用。
- 置顶层级使用 `screen-saver`，以尽量减少被其他窗口压住的概率。

### Linux (X11)

- 基础功能可用：连接、模型渲染、历史记录、截图、录音快捷键、打包发行。
- 智能穿透不可用（当前平台无法稳定转发鼠标事件）。
- 全屏应用检测使用 `active-win` 的启发式判断，准确度取决于桌面环境和窗口管理器。
- 自动更新不可用，需要用户手动下载新版本。

### Linux (Wayland)

- 核心功能可用：连接、模型渲染、历史记录、截图、录音快捷键、打包发行。
- 智能穿透不可用（当前平台无法稳定转发鼠标事件）。
- 自动检测全屏应用不可用，因为当前实现无法稳定获取活跃窗口边界。
- 自动更新不可用，需要用户手动下载新版本。
- 这是当前跨平台体验差异最大的运行环境，建议在用户反馈异常时优先确认是否为 Wayland 会话。

## 运行时提示策略

推荐在以下情况下主动提示用户：

- `Linux (Wayland)`：提示“智能穿透”和“全屏应用检测”不可用。
- `Linux (X11)`：提示“智能穿透”不可用，“全屏应用检测”为启发式能力。
- `Windows arm64`：提示“全屏应用检测”已禁用，但其他能力正常。

## 排障建议

- 用户反馈模型无法自动避让全屏窗口时，先确认平台与会话类型。
- 用户反馈点击穿透或窗口前后台异常时，优先区分 `Windows/macOS` 与 `Linux`。
- Linux 环境优先在 X11 会话下验证；Wayland 需接受部分能力降级。
- 若打包后原生模块异常，优先检查目标平台是否完成对应的 Electron 原生依赖重建。
